（前几天推荐的 listary 受到了感谢，他说多亏了你，完全想不到能够这样管理文件，我说能搜索先搜索，就算没有索引计算机也会比你看得快；随后谈到我们应该怎样让计算机变成广义上的 enhance tool，得到了一些简单的共识）

（审慎的说法是，因为我们目前还没有被某些软件提供的搜索功能坑过，所以我们可能会越来越依赖它，以至于深信不疑，这样一旦以后出了问题，我们也就难以第一时间发现它了；我们同时可能被稳定准确的搜索结果宠坏，忽视了搜索出错的可能）
（但是搜索的内在需求，并不因为这些问题而减弱、变质）

（模仿是必然陷入被动的，正所谓（如愚见指月，观指不观月），他今天知道了一个 listary 好用又如何，如果想做一批文档内的文字搜索，是不是又傻眼了；想要批量修改搜索到的文档内的文字，而后再进行搜索，是不是又不行了…）

（就有人会觉得，为什么你现在就会考虑这么多，这就是工程人的可悲之处，在项目的进程中的你是容不得好奇的，就比如你正查着一个小问题的解法呢，突然想起（ vscode 是不是应该换换主题和背景色）（环境变量好乱啊要不重装一遍系统吧）（要不我写个脚本专门做这个事吧）…那你这项目还做不做了…）
（所以你平时就一定要知道有什么不舒服、看不顺眼的地方，早点改掉；有什么（以为不需要，实际是自己不会）的东西，尽快学清楚…）
（人们往往高估自己在设备上的需求度，低估自己技能上的需求度，很多技能你是不会而不是不需要，而很多设备则相反…你不可能需要你本身不会的东西，盲点也就此产生）
（直观感受有时是靠谱的，比如你不懂核物理，然后觉得那些知识一辈子用不上因此是不需要的，这点确实不假；但（因为不懂所以不需要）的这种逻辑是不太靠谱的，所以至少所在行业内的知识，是一定要尽量精通、透彻，做到没有明显的疑点才行）

（如果他勤奋一点，ok好的那我用 filelocator pro 类似的东西去实现文档内搜索，批量修改的话用 vscode 内置的跨文件批量删改，这样总行了吧…这样确实算很可以了，但还是差最后的一点火候）
（你和你的需求之间，差的难道仅是一个软件，一个所谓search and replace utility 吗）

（就他最后应该想到 selected fields, query statements…然后想到 api endpoints 之类的 数据即服务 的东西，然后想到一些基础的范例，比如说这个）

（讲到这里基本大同小异了，讲真还是看你怎么去认识和抽象一些特定的需求，都是成年人了）

（就我猜有人会问，如果他搞一手（弱者道之用），白嫖我这一堆经验方法，然后加以改良或直接兜售，用他的名义发布了，那怎么办）
（首先这并非独门的经验方法，而是我不知从何处看来的，忘了具体哪里反正不是我自己想到的；再者，这东西如果还能算作经验方法的话，我觉得大家可能是太热衷于赞美了…）

（不用怀疑，我们能活到现在，肯定已经点满了幸运和无耻的天赋了，要不然我们纵不横死，也要因一事无成而羞愤自杀…所以这两方面是具足的，而次序的话，则是（因过于无耻而不自杀是幸运的一部分，正因如此才使得幸运的我们更加无耻）…）

（形而上的讨论当然过瘾，但现实总是芜杂，以此推得形而上世界里的形而上在俯瞰众生时，一定也会有相似的芜杂感…于是每天我都要看一两遍关于蛆虫和死尸的影片，想到自己这么干净，自信和安心感就油然而生）

（你看看我，一边能说一套套切中肯綮的方法论颠倒众生，一边又能在几行锲阔谈宴里让人们难受得食欲不振…这就是动人心魄的甜美的（控制）啊，方法又算什么，（弱者道之用）又算什么…看看我，请认认真真的看看我吧，我正在孵化更多的王虫）

（过奖了，一点微不足道的才能而已…我想人们会觉得道德更贴身，而法律更有效，所以背弃道德而坚守法律的人大概会很卑鄙，然后背弃法律而坚守道德的人大概会很高尚，这就是高尚者和卑鄙者的区分，卑鄙者是最懂心机的…）

（并不，正是因为卑鄙者处处得势，处处占尽先机，卑鄙才会最终失败…并不，只要卑鄙没有占尽先机，它就必然向着占尽优势、占尽先机的方向有力地坚实地前进，因为只有到它最先进的那一刻，永恒才决定它的功绩归零…不需要赢，如果赢的概念还存在那必然就是大卑鄙取代小卑鄙的循环，没有突破的…）

（愚公正是因为自己这辈子乃至子子孙孙都移不走山，才说明愚公的正确：只有你相信、并且愿意把生命交给无尽的时间去审判，只有当你相信无尽的时间可以攘除掉所有的善恶之辨，输赢之分，山才会因空间时间的不连续性被击倒，因为高阶来看变成了无穷小嘛…）
我想要现实点的答案
（那就是永远斗下去…）
